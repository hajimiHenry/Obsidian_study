# 一阶逻辑推理 · 实操套路

> 一阶逻辑证明比命题逻辑多了一组量词规则（∀-、∀+、∃+、∃-），但**套路反而更固定**。大部分题就一条主线来回走。

---

## 四条量词规则速查

| 规则 | 写法 | 方向 | 含义 |
|---|---|---|---|
| ∀- | $\forall x\,A(x) \vdash A(t)$ | 拆全称 | 「所有都满足」→「某个具体对象也满足」 |
| ∃+ | $A(c) \vdash \exists x\,A(x)$ | 组存在 | 「某个对象满足」→「存在对象满足」 |
| ∃- | $\exists x\,A(x)$ → 引入新常项 $c$，得 $A(c)$ | 拆存在 | 给那个「存在但未知」的对象起个临时名字 |
| ∀+ | $A(y)$（$y$ 是任意的）$\vdash \forall x\,A(x)$ | 组全称 | 「任意对象都满足」→「所有对象都满足」 |

**记法**：$\forall$ 和 $\exists$ 各有一拆一组，拆在前、组在后，凑成最常见的证明路线。

---

## 核心套路：一阶逻辑证明的万能路线

99% 的一阶逻辑证明题都走下面这条线：

$$
\exists\text{前提} \ \xrightarrow{\exists-}\ \text{具体对象}\ \xrightarrow{\forall-}\ \text{套全称性质}\ \xrightarrow{\text{命题逻辑推理}}\ \text{具体结论}\ \xrightarrow{\exists+}\ \exists\text{结论}
$$

一句话：**先拆存在（拿到一个具体的人），再拆全称（把通用规律套到这个人身上），推完以后再组回存在。**

---

## 四种最常见的题型

### 题型一：全称 + 具体事实 → 具体结论

**特征**：前提有一个 $\forall$ 和一个关于常项的事实，结论也是关于那个常项。

**套路**：对 $\forall$ 做 ∀-（把 $x$ 换成那个常项），然后命题逻辑接上。

**例题**：$\{\forall x\,(M(x) \to N(x)),\ M(a)\} \vdash N(a)$

| 步骤 | 公式 | 依据 |
| :-: | :--- | :--- |
| ① | $\forall x\,(M(x) \to N(x))$ | 前提引入 |
| ② | $M(a)$ | 前提引入 |
| ③ | $M(a) \to N(a)$ | ① ∀- |
| ④ | $N(a)$ | ②③ 假言推理 |

---

### 题型二：全称 + 存在前提 → 存在结论（最常考！）

**特征**：前提里有一个 $\exists$ 和一个 $\forall$，结论也是 $\exists$。

**套路**：先 ∃- 取对象 → 再 ∀- 套规律 → 推 → 最后 ∃+ 组回去。

**例题**：$\{\forall x\,(P(x) \to Q(x)),\ \exists x\,(P(x) \land R(x))\} \vdash \exists x\,(Q(x) \land R(x))$

| 步骤 | 公式 | 依据 |
| :-: | :--- | :--- |
| ① | $\forall x\,(P(x) \to Q(x))$ | 前提引入 |
| ② | $\exists x\,(P(x) \land R(x))$ | 前提引入 |
| ③ | $P(c) \land R(c)$ | ② ∃-（引入新常项 $c$） |
| ④ | $P(c)$ | ③ 化简式 |
| ⑤ | $R(c)$ | ③ 化简式 |
| ⑥ | $P(c) \to Q(c)$ | ① ∀- |
| ⑦ | $Q(c)$ | ④⑥ 假言推理 |
| ⑧ | $Q(c) \land R(c)$ | ⑤⑦ 合取 |
| ⑨ | $\exists x\,(Q(x) \land R(x))$ | ⑧ ∃+ |

**心法**：注意顺序——**一定要先 ∃- 再 ∀-**。因为 ∀- 需要一个具体对象来代入，而这个对象正好由 ∃- 提供。反过来做会卡住。

---

### 题型三：结论是蕴含式（一阶版 CP）

**特征**：结论是 $\exists x\,A(x) \to \exists x\,B(x)$ 这种带 $\to$ 的形状。

**套路**：跟命题逻辑一样用附加前提法——先加前件，在里面走一遍「∃- → ∀- → 推 → ∃+」的流程，最后撤去。

**例题**：$\{\forall x\,(F(x) \to G(x))\} \vdash \exists x\,F(x) \to \exists x\,G(x)$

| 步骤 | 公式 | 依据 |
| :-: | :--- | :--- |
| ① | $\forall x\,(F(x) \to G(x))$ | 前提引入 |
| ② | $\exists x\,F(x)$ | 附加前提 |
| ③ | $F(c)$ | ② ∃-（引入新常项 $c$） |
| ④ | $F(c) \to G(c)$ | ① ∀- |
| ⑤ | $G(c)$ | ③④ 假言推理 |
| ⑥ | $\exists x\,G(x)$ | ⑤ ∃+ |
| ⑦ | $\exists x\,F(x) \to \exists x\,G(x)$ | 撤去附加前提 ② |

**心法**：一阶 CP 和命题 CP 的区别只在于——附加前提之后的那段推理里多了量词操作。

---

### 题型四：多重量词前提

**特征**：前提里有 $\forall x\forall y$ 或者 $\forall x\exists y$ 这种多层量词。

**套路**：从最外层量词开始，一层一层 ∀- 剥开。每次 ∀- 只能剥最外面那层。

**例题**：$\{\forall x\forall y\,(P(x,y) \to Q(x,y)),\ P(a,b)\} \vdash Q(a,b)$

| 步骤 | 公式 | 依据 |
| :-: | :--- | :--- |
| ① | $\forall x\forall y\,(P(x,y) \to Q(x,y))$ | 前提引入 |
| ② | $P(a,b)$ | 前提引入 |
| ③ | $\forall y\,(P(a,y) \to Q(a,y))$ | ① ∀-（$x$ 换 $a$） |
| ④ | $P(a,b) \to Q(a,b)$ | ③ ∀-（$y$ 换 $b$） |
| ⑤ | $Q(a,b)$ | ②④ 假言推理 |

---

## 做题时的决策流程

```
1. 结论里有 → 吗？
   → 有：附加前提法，进到里面再做量词操作
   → 没有：往下看

2. 前提里有 ∃ 吗？结论是 ∃ 吗？
   → 是：走万能路线（先 ∃-，再 ∀-，推，最后 ∃+）
   → 不是：往下看

3. 前提只有 ∀，结论是原子公式？
   → 直接 ∀- 代入具体常项，然后命题逻辑收尾
```

---

## ∃- 的坑（最容易错的一条规则）

∃- 有三个铁律：

1. **新常项必须是全新的**——不能是前提里已经出现过的 $a,b,c$。如果你已经用过 $c$，下次 ∃- 要用 $d$。
2. **新常项只是临时代表**——你不能偷偷给它加额外性质（比如不能因为「存在一个人」就假设「这个人是男人」）。
3. **∃- 要趁早做**——通常在有多个前提时，先处理 ∃ 前提，拿到具体对象，再去 ∀ 前提里套。

---

## ∀+ 的坑（最容易被忽略的一条）

∀+ 的意思是：如果你证明了对**任意一个** $y$ 都成立，那才对**所有** $x$ 成立。

关键条件：**$y$ 不能出现在任何未撤去的附加前提中**。也就是说，你证明 $A(y)$ 的时候，$y$ 必须真的是"任意的"——没有偷偷依赖任何假设。

实际考试里 ∀+ 出现得不多，大部分题最后一步是 ∃+ 而不是 ∀+。

---

## 跟命题逻辑证明的关系

一阶逻辑证明 = 命题逻辑证明 + 量词拆装。

拿到题以后：
1. 先看量词结构，决定 ∃- 和 ∀- 的先后顺序
2. 量词处理完后，剩下的就是纯命题逻辑推理
3. 最后如果需要，再 ∃+ 或 ∀+ 把量词装回去

**别被量词吓住**——它们就是"拆包裹"和"封包裹"两步，中间那段还是你会的命题逻辑。
